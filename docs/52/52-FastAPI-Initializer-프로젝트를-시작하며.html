SpringBoot, Java 로 개발을 해오다가 이번에 회사에서 FastAPI, Pytohn 으로 전환을 하게 되었다. 그동안 Java 와 Spring 프레임워크를 공부해놓은게 아쉽게도 했지만 원래 Python 에 관심이 많기도 했으며, 전 세계적으로 Python 이 대세가 되는것 같아 시대의 흐름에 몸을 맡기기로 했다.

 

그리고 FastAPI 를 이용하다보니 불편한것은 딱 하나였다.

프로젝트를 새롭게 세팅할때마다 새롭게 가상환경과 의존성을 세팅해줘야한다는 것이다. 특정 IDE 를 사용하면 자동으로 된다지만...그냥 무료버전에서 Spring initializer 를 이용하던 나에게는 여간 불편한게 아니었다.

 

그래서 Spring initializer 같은걸 하나 만들기로 마음 먹었다. 중요한 아이디어는 다음과 같았다.

(1) PyPI 에서 원하는 패키지를 읽어와서 세팅해주고

(2) 가상환경 세팅도 해주고

(3) 최종적으로 보일러 플레이트가 세팅된 FastAPI 프로젝트 압축 파일을 사용자에게 내려주는 것

 

여기에 약간 조미료를 더해서 우리 회사 프론트 엔지니어분들이 사용하는 next.js * TypeScript 를 이용해보리고 마음 먹었다.

 

대략 5시간정도 고생을 한 끝에 첫번째 버전을 릴리즈 할 수 있었다.

 

가비아에서 마침 딱 좋은 도메인은 6000 원에 파는 이벤트를 하고 있길래 ... https://www.fastapi.my 도메인에 vercel 을 이용해 세팅하였다.
