문제

N 명의 사람들이 한줄로 서 있다. 그들이 줄 서 있는 순서대로 N 개의 키가 주어진다. 이때 줄 서 있는 사람들이 '서로를 볼 수 있는 쌍'의 개수를 구하여라 (서로를 볼 수 있다는 것은 둘 사이에 둘 중 누구하나보다 키가 큰 사람이 없다는 것을 의미한다.)

시간 제한: 1초

데이터 크기:  1 <= N < 500,000

예시

이 문제는 본문만 읽으면 조금 헷갈려서 예시를 제대로 짚고 넘어가겠다.

사람들이 줄을 서 있고 그 키가 배열 [2, 4, 1, 2, 2, 5, 1] 로 주어진 예시를 생각해보자

{0번째, 1번째} 사람은 서로를 볼 수 있지만 {0번째, 2번째} 사람은 중간에 1번째 사람이 키가 4로 {0번째, 2번째} 둘 모두 보다 크기 때문에 서로 볼 수 없다.

 

이러한 논리로 좌측에서부터 우측으로 이동하면서 서로 관측가능하고 중복되지 않는 좌측 키 <-> 우측 키 쌍을 정리해보면 아래와 같다.

  • i=0, 2<->4
  • i=1, 4<->1, 4<->2, 4<->2, 4<->5
  • i=2, 1<->2
  • i=3, 2<->2, 2<->5
  • i=4, 2<->5
  • i=5, 5<->1
  • i=6, 중복되지 않는 A<->B 관측 케이스가 없음.

따라서 [2, 4, 1, 2, 2, 5, 1] 의 줄에서는 총 10개의 서로 관측 가능한 쌍이 존재한다. 

 

 

출처: https://www.acmicpc.net/problem/3015

접근

부르트 포스

시간제한 1초, 데이터 크기 50만이라는 점에서 시간복잡도 O((n*n)) 가 기대되는 부르트 포스 접근은 요건을 충족할 수 없다.

 

Stack 을 이용한 접근

솔직히 백준 사이트에서 Stack 을 이용한 문제라는 힌트가 있었기 때문에 고민이라도 시작할 수 있었다. 이런 힌트조차 없었다면 정말 어려웠을것 같다.

 

기본적인 아이디어는 "Stack 안에 키를 내림차순으로 유지" 해야 한다는 것이다. 위의 예시였던 [2, 4, 1, 2, 2, 5, 1] 를 Stack 에 넣고 계산하는 과정을 다음의 그림의 형식을 이용해서 설명하겠다.

 

2는 최초에 있었고, 4가 새롭게 들어오려고 한다. Stack 은 더이상 내림차순이 아니므로 Stack 이 내림차순이 될때까지 pop 을 하면서 카운터를 증가시켜준다.
